# Spec & Plan：investment-h5 资产配置 / 加仓决策功能

## 一、目标

为 `investment-h5` 增加一套“资产配置 + 新增资金决策”能力，帮助用户把“以后新增资金该买什么”从临时拍脑袋，升级为**有目标、有偏离监控、有建议动作**的系统化规则。

核心目标：
1. 用户可配置目标资产占比，如：股票 / 债券 / 宽基 / QDII / 固收。
2. 系统基于当前持仓，自动计算实际占比与目标占比偏离。
3. 当用户输入“本次新增资金金额”后，系统给出建议投向。
4. 支持规则化约束，避免随意加仓，例如：
   - 优先补低配资产
   - 单只基金不超过某个上限
   - 先大类，后细分，最后落到具体基金
5. 结果要适合手机端快速查看，并能解释“为什么这样分配”。

## 二、当前项目上下文（已确认）

- 前端：Vue 3 + Vite
- 当前已有页面：`Positions.vue`、`Stats.vue`、`PositionDetail.vue`
- 当前已有数据能力：
  - 持仓列表 `positionApi.list()`
  - 统计总览 `statsApi.overview()`
  - 历史统计 `statsApi.history()`
  - 基金详情 `fundApi.detail()`
- 当前前端已存在基金类型筛选概念，但“资产配置/目标占比/加仓建议”能力尚未存在。

## 三、我先做的关键假设

1. 本功能先服务于**个人投资决策**，不是多人协同审批系统。
2. 第一阶段先按**前端可落地 + 现有持仓数据可计算**设计，不强依赖复杂后端模型。
3. 目标配置分两层：
   - 大类资产层：股 / 债 / 固收 / QDII / 现金（可选）
   - 投资标的层：具体基金
4. 第一阶段建议只处理“新增资金如何分配”，**不自动处理卖出再平衡**。
5. 第一阶段不做收益预测模型，只做**规则驱动型建议**，保证可解释、易维护。

## 四、功能拆分建议

### Phase 1：资产配置看板（先看清）

目标：先把“现在的仓位结构”展示清楚。

功能：
- 资产配置概览卡
  - 当前总资产
  - 各大类当前金额 / 当前占比
  - 目标占比
  - 偏离值（超配 / 低配）
- 偏离列表
  - 低配优先显示在前
  - 红/绿或强调色标识偏离方向
- 配置规则入口
  - 编辑目标占比
  - 编辑分类映射

建议数据结构：
- `allocationTargets`
  - `asset_key`
  - `asset_name`
  - `target_pct`
  - `enabled`
- `positionClassifications`
  - `fund_code`
  - `asset_key`
  - `sub_asset_key`
  - `manual_override`

### Phase 2：新增资金决策器（开始能用）

目标：输入一笔新增资金，直接得到买入建议。

功能：
- 输入本次新增资金金额（如 1000 / 3000 / 10000）
- 系统计算：
  - 若只按大类补低配，应分到哪些资产类别
  - 若要落到具体基金，应优先哪些基金
- 输出建议：
  - 建议方案摘要
  - 各资产类别建议金额
  - 各基金建议金额
  - 本轮执行后预计占比变化

建议规则（v1）：
1. 先看大类低配程度
2. 新增资金优先补低配资产
3. 如果某大类下有多只基金：
   - 优先补当前占比低的核心标的
   - 或用户手动指定“默认承接基金”
4. 若某基金达到单基金上限，则停止继续分配到该基金
5. 金额按 10 元或 100 元粒度取整（后续可配置）

### Phase 3：规则系统（规范自己）

目标：把“原则”写成可执行规则。

建议规则维度：
- 目标配置规则
  - 各类目标占比合计必须为 100%
- 风险边界规则
  - QDII 不超过 X%
  - 单只基金不超过 X%
  - 单账户不超过 X%
- 加仓优先级规则
  - 先补债，再补股
  - 宽基优先于行业主题
  - QDII 仅在低于目标 X% 时才加
- 交易约束规则
  - 单次最小买入额
  - 单次最多推荐标的数
  - 保留现金缓冲（可选）

### Phase 4：执行留痕（闭环）

目标：让建议和实际操作形成闭环。

功能：
- 保存一次“加仓建议快照”
- 标记是否已执行
- 已执行后可一键跳转到新增交易/持仓页面
- 后续可对比：
  - 建议 vs 实际执行
  - 执行后仓位是否更接近目标

## 七、建议算法（v1）

### 输入
- 当前持仓列表
- 当前分类后的资产金额
- 目标占比
- 新增资金金额
- 规则约束

### 计算步骤
1. 统计当前各资产类别金额与占比
2. 计算目标金额 = （当前总资产 + 新增资金）× 目标占比
3. 计算各类缺口 = 目标金额 - 当前金额
4. 仅对缺口 > 0 的类别分配新增资金
5. 若总缺口超过新增资金，则按缺口比例分配
6. 每个类别内再落到具体基金：
   - 优先默认基金
   - 否则按“当前占比低 / 当前金额小 / 核心标的优先”分配
7. 应用约束：
   - 单基金上限
   - 最小下单金额
   - 推荐标的数上限
8. 输出最终建议清单

### 输出示例
- 本次新增资金：3000 元
- 建议：
  - 债券：1200
    - XX纯债A：1200
  - 宽基：1000
    - 沪深300ETF联接A：600
    - 中证红利A：400
  - QDII：800
    - 标普500QDII：800
- 执行后预计配置：
  - 债券 39.2% → 40.0%
  - 宽基 22.5% → 23.0%
  - QDII 11.8% → 12.5%

## 十四、根据最新确认收敛后的第一版页面方案（2026-06-24）

### 1. 页面定位

第一版确定为：**独立资产配置页面 `Allocation.vue`**。

这个页面不再先挂统计页，而是作为独立能力存在，职责是：
1. 创建资产配置方案
2. 维护资产配置目标比例与允许偏差
3. 从当前持仓中挑选基金加入该配置方案
4. 维护基金在方案内的状态（保留 / 观察 / 可调出 / 禁买）
5. 观察各类别与各基金是否超标 / 低配
6. 在配置方案内部直接生成新增资金配置建议与调仓提示

### 2. 第一版表单字段（用户已明确）

用户可创建**多个资产配置方案**。

每个资产配置方案创建时，先填写 6 类基金的目标比例：
- 纯债基金
- 固收类基金
- 红利类基金
- 指数类基金
- 海外 QDII 基金
- 其他基金

并且每一类都要额外填写：
- 目标比例
- 允许偏离百分比（最大偏差）

例如：
- 纯债基金：目标 30%，允许偏离 ±5%
- 海外 QDII：目标 15%，允许偏离 ±3%

基础校验：
- 各项比例不能为空
- 每项比例 >= 0
- 每项最大偏差 >= 0
- 总和必须 = 100%

建议补充字段：
- 配置方案名称（如：稳健配置 / 长期定投配置）
- 备注（可选）
- 是否默认方案（后续扩展）
- 排序权重（后续支持多方案排序时可用）

### 3. 第一版页面主结构

#### 区块 A：配置方案表单
- 配置名称
- 六类基金目标比例输入框
- 实时显示“当前合计 = xx%”
- 合计不是 100% 时禁止保存

#### 区块 B：从当前持仓添加基金
- 拉取当前持仓列表
- **由用户手动选择**要纳入当前配置方案的基金，不做默认全部纳入
- 每只基金要有：
  - 基金名称
  - 基金代码
  - 所属账户
  - 当前市值
  - 归属类型（六选一，可自动识别 + 手动改）
- 唯一性约束：
  - **同一个账户下的同一支基金，只允许出现在一个资产配置方案里**
  - 唯一键建议按 `account_id + fund_code` 处理
  - 若该基金已被其他方案占用，当前页面应禁选或明确提示“已纳入方案 X”

#### 区块 C：资产配置统计
- 按六类展示：
  - 目标比例
  - 允许偏离百分比
  - 当前比例
  - 偏离值
  - 状态（达标 / 低配 / 超配）
- 判断规则：
  - 若 `当前比例 < 目标比例 - 允许偏差`：低配
  - 若 `当前比例 > 目标比例 + 允许偏差`：超配
  - 其余：达标
- 用颜色区分状态：
  - 超出允许区间：红/绿强调
  - 落在允许区间内：正常

#### 区块 D：配置内基金明细
- 该配置中包含的基金列表
- 每只基金显示：
  - 当前市值
  - 占整个配置的比例
  - 占所属类别的比例
  - 基金状态：保留 / 观察 / 可调出 / 禁买
  - 是否超过单基金合理上限（若启用规则）
- 基金状态在建议系统中的含义：
  - **保留**：正常持有，可继续作为配置内核心标的
  - **观察**：可继续持有，但新增资金建议应降低优先级
  - **可调出**：当类别超配或需要腾挪时，优先提示为调仓候选
  - **禁买**：禁止把新增资金继续配置到该基金

#### 区块 E：方案内配置建议
- 在当前配置方案内部直接生成建议，不用跳到别的页面
- 建议分两类：
  1. **新增资金配置建议**：告诉用户新增资金优先补到哪些类别 / 哪些基金
  2. **调仓提示**：告诉用户哪些类别超配、哪些基金更适合作为调出候选
- 建议输出应包含：
  - 建议原因
  - 建议目标类别
  - 建议具体基金（若能落到基金层）
  - 受基金状态字段影响的说明

建议规则（v1）示例：
- 新增资金优先流向**低配且未超过允许偏差下限**的类别
- 类别内优先分配给状态为 **保留** 的基金
- 状态为 **观察** 的基金降低优先级
- 状态为 **禁买** 的基金不得进入新增资金建议
- 当类别超配时，状态为 **可调出** 的基金优先进入调仓提示
- 若某类别下没有可买基金（例如全是禁买/可调出），则只给出类别级建议，不强行落到基金级

### 4. 第一版最值得补强的优化点（建议优先做）

#### 优化 1：自动分类 + 手动覆盖
这是必须做的，不然用户每次手动分类型会很累。

建议：
- 先根据基金名 / 已有分类规则自动猜测类别
- 用户可以手动改
- 手动修改后记住 fund_code → asset_type 的映射

这样后面新增同一只基金时，不用重复选择。

#### 优化 2：实时校验 + 视觉反馈
用户填比例时，不能只在点保存后才报错。

建议：
- 输入时实时显示总和
- 若 < 100%，显示“还差 xx%”
- 若 > 100%，显示“超出 xx%”
- 保存按钮自动禁用

手机端体验会好很多。

#### 优化 3：超配 / 低配阈值不要只靠颜色
只用颜色不够，尤其手机上不一定一眼看懂。

建议同时显示状态文案：
- 严重低配
- 轻微低配
- 达标
- 轻微超配
- 严重超配

阈值示例：
- 偏离绝对值 <= 2%：达标
- 2% ~ 5%：轻微偏离
- > 5%：明显偏离

#### 优化 4：手动选择纳入基金 + 占用冲突提示
既然你已经明确要求“用户手动选择纳入”，那这一块的重点就不再是一键导入，而是**选择效率和冲突反馈**。

建议增加：
- 按账户分组展示当前持仓
- 支持勾选加入当前配置方案
- 对已被其他方案占用的基金直接禁选
- 同时展示占用提示：`已纳入：稳健配置` 

这样用户不会选完后才发现冲突。

#### 优化 5：区分类别占比与单基金占比
你刚才提到“资产配置统计用户添加的基金占当时资产配置的比例”，这里建议分成两层，不要混在一起：

- 第一层：**类别占比**
  - 例如 QDII 当前 18%，目标 15%
- 第二层：**单基金占比**
  - 例如 标普500 当前占整个配置 9%

这样用户会更容易知道：
- 是某个大类超了
- 还是某只基金太重了

#### 优化 6：预留“默认承接基金”能力
你现在已经明确要求“方案内直接给配置建议”，所以这个能力从“预留”升级成“建议尽早落地”。

建议在每个类别里预留一个字段：
- 默认承接基金

例如：
- 指数类默认承接：沪深300
- QDII 默认承接：标普500

这样新增资金建议在落到基金层时会更稳定、更符合你的真实偏好。

同时要结合基金状态字段一起判断：
- 默认承接基金如果状态是 **禁买**，则必须跳过
- 默认承接基金如果状态是 **观察**，则降级为次优先
- 默认承接基金如果状态是 **可调出**，则不应用于新增资金建议

#### 优化 7：配置快照时间要固定
因为你说的是“用户添加基金到资产配置中，统计当时持有基金占当时资产配置的比例”。

这里建议非常明确地区分：
- **当前实时配置**
- **创建当时快照**

第一版如果不做历史快照，至少要在页面上明确：
- 当前占比是**按实时持仓市值**计算

否则以后你可能会误以为这是“创建那一刻”的静态结果。

### 5. 我建议第一版额外增加的几个小字段

为了后续扩展更稳，我建议配置项里顺手加上：
- `name`：方案名
- `created_at`
- `updated_at`
- `included_fund_codes`：纳入该配置的基金
- `fund_type_overrides`：手动改过的基金分类
- `fund_status_map`：基金状态映射（保留 / 观察 / 可调出 / 禁买）
- `default_fund_by_type`：各类别默认承接基金

如果先走前端本地存储版，这些也建议结构先定好。

### 6. 第一版暂时不要做的内容

为了避免过重，先不做：
- 多个配置方案之间的对比
- 历史配置快照回放
- 自动执行调仓
- 卖出交易自动生成
- 收益回测
- 复杂风险评分

但**方案内建议**（新增资金建议 + 调仓提示）现在已经进入第一版范围。

### 7. 第一版 MVP 我建议的最终范围

最小可上线版本建议只做：
1. 新建一个独立资产配置页面
2. 支持创建 / 编辑 / 切换多个资产配置方案
3. 为六类基金分别填写目标比例 + 允许最大偏差
4. 用户手动从当前持仓中选择纳入基金
5. 自动 / 手动设置基金类别
6. 为纳入基金设置状态：保留 / 观察 / 可调出 / 禁买
7. 同账户下同基金禁止出现在多个方案中
8. 展示类别当前占比 vs 目标占比 vs 允许偏差
9. 展示基金在配置中的占比与状态
10. 在方案内部生成新增资金建议
11. 在方案内部生成调仓提示
12. 用颜色 + 文案提示是否超标 / 低配 / 达标

这样已经足够让你开始“规范新增资金的决策前提”，并且能真正给出贴合你习惯的建议。

### 8. 下一阶段自然延伸

等第一版跑通后，下一步就可以直接接：
- 输入新增资金金额
- 按低配类别自动分配
- 类别内按默认基金或低占比基金承接
- 输出买入建议清单
