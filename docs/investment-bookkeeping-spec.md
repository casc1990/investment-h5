# 投资收益记账系统规格说明（基于 investment-h5 扩展）

## 目标
在现有 `investment-h5` 项目基础上，扩展成一个适合你长期使用的 **投资收益记账系统**，重点解决：

1. 记录基金/账户/成员维度的持仓与收益
2. 每日同步基金净值并追踪 **昨日涨跌 / 持有收益 / 累计收益**
3. 支持按账户、成员、基金查看收益表现
4. 形成可长期维护的收益台账，而不仅是“当前持仓展示”

系统优先面向你的当前场景：
- 主要资产：基金
- 使用方式：手机 H5
- 数据来源：东方财富 / 天天基金公开数据 + 你手工录入的持仓/历史收益
- 部署方式：继续沿用现有 `investment-h5` 的前端 H5 + API / D1 / 定时同步思路

---

## 我当前基于代码做出的判断

### 已存在能力
从当前代码可见，项目已经具备这些基础：
- 登录鉴权
- 账户管理（accounts）
- 成员管理（members）
- 持仓管理（positions）
- 交易记录（trades）
- 行情同步（market / fund sync）
- 统计总览（stats）

### 还缺的“记账系统”能力
要成为真正好用的投资收益记账系统，还需要补齐：
- **每日收益快照**：每天自动保存每只基金/每个账户/全组合的收益数据
- **收益走势图 / 时间序列**：不是只看当前值，要能看近7天、30天、累计变化
- **交易对收益的影响拆分**：区分“净值涨跌带来的收益”与“加仓/减仓带来的资产变化”
- **台账视角**：能按日生成历史记录，方便回溯
- **组合分析**：各基金占比、收益贡献、账户对比、成员对比

## 假设
如无特别说明，我先按以下假设设计：

1. 继续在 **现有 `investment-h5` 项目** 上迭代，而不是新开一个项目
2. 继续优先支持 **基金**，后续可再扩展股票/现金/债券等资产
3. 收益计算以“持仓 + 每日净值快照”为核心，不先做复杂税费/分红再投资自动核算

## 功能规格

### Phase 1：MVP（优先做）

#### 1. 持仓记账
- 录入基金代码、基金名称、份额、买入成本、初始累计收益、账户、成员、分红方式
- 展示当前市值、昨日收益、持有收益、收益率
- 手动同步单只 / 一键同步全部净值

#### 2. 每日收益快照
系统每天自动保存以下数据：
- 每只持仓的：
  - 基金代码
  - 日期
  - 净值
  - 市值
  - 昨日收益
  - 持有收益
  - 持有收益率
- 每个账户汇总：
  - 总资产
  - 昨日收益
  - 持有收益
- 全组合汇总：
  - 总资产
  - 昨日收益
  - 持有收益
  - 累计收益

#### 3. 收益台账页
新增一个“收益台账”页面：
- 查看最近 7 / 30 / 90 天组合收益
- 按日查看总资产、昨日收益、累计收益
- 点击某天可展开看持仓明细

#### 4. 基础统计分析
在统计页增强：
- 近7天收益趋势
- 本月累计收益
- 各基金收益贡献排名
- 各账户收益贡献排名

---

### Phase 2：增强版
- 交易流水和收益拆分（加仓/减仓/分红/转换）
- 账户净流入、净流出、已实现收益
- 成员维度分析
- 基金分类分析（QDII / 债基 / 指数 / 红利）
- 收益日历 / 月度报表

## 边界
### Always
- 保持手机端可读性优先
- 先兼容你当前的基金持仓场景
- 所有收益计算公式要可复核

### Ask First
- 修改数据库表结构
- 改动现有收益计算口径
- 新增第三方依赖

### Never
- 不在未确认前直接改掉你现有线上数据
- 不把临时推断当成真实收益写入正式台账
